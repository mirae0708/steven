# AI Council — 분석 방법론

## 개요
AI Council은 3명의 AI 분석가가 독립적으로 시장을 분석하고 다수결로 매매 결정을 내리는 시스템이다. 각 분석가는 서로 다른 관점과 도구를 사용하여 평가하며, 최종 결정은 3인의 합의 또는 다수결로 이루어진다.

## 분석가 구성

### 📊 기술적 분석관 (Technical Analyst)
**역할**: 차트 패턴, 기술적 지표 기반 분석  
**평가 기준**:
- 이동평균선 (20일/60일/120일)
- RSI (과매수/과매도)
- 볼린저밴드 (변동성)
- 거래량 추세
- 지지/저항 레벨
- MACD (추세 모멘텀)
- Stochastic (추세 전환)

**판단 스타일**: 보수적 — 추세 확인 후 진입, 기술적 패턴 중시

### 🔍 뉴스 모멘텀 분석관 (News Momentum Analyst)
**역할**: 뉴스 감성, 키워드 트렌드 기반 분석  
**평가 기준**:
- 뉴스 헤드라인 감성 분석 (긍정/부정/중립)
- 업종별 뉴스 모멘텀
- 키워드 빈도 변화
- 시장 심리 지표
- 기업 공시/이슈

**판단 스타일**: 공격적 — 뉴스 모멘텀 포착 시 선제 대응

### 📉 리스크 관리관 (Risk Manager)
**역할**: 포트폴리오 리스크, 시장 환경 평가  
**평가 기준**:
- CB Score (Circuit Breaker Score)
- 포트폴리오 손익률
- 종목별 비중 한도
- 시장 변동성 (VIX 대체 지표)
- 상관관계 리스크
- 환율/원자재 영향

**판단 스타일**: 방어적 — 리스크 우선, 손절 규칙 엄격

## 의사결정 프로세스

### 1. 데이터 수집
- market_intel.json에서 시장 데이터 로드
- 각 분석가별 필요 데이터 추출
- 포트폴리오 현황 로드

### 2. 독립 분석
각 분석가가 동시에 독립적으로 분석 수행:
```
[기술적 분석관] → BUY/SELL/HOLD + 근거
[뉴스 분석관]   → BUY/SELL/HOLD + 근거
[리스크 관리관]  → 포지션 사이즈 + 리스크 평가
```

### 3. 다수결 투표
| 투표 결과 | 결정 |
|:---------|:------|
| 3 BUY | 강력 매수 (CB 허용 범위 내 최대) |
| 2 BUY 1 HOLD | 매수 (중간 사이즈) |
| 2 BUY 1 SELL | 매수 (소형) |
| 모두 HOLD | 유지 (현상 유지) |
| 2 SELL 이상 | 매도/포지션 축소 |

### 4. 포지션 사이징
CB Score 기반으로 최종 사이즈 결정:
| CB Score | 포지션 사이즈 |
|:---------|:-------------|
| 0-20 (극단 공포) | 최대 20% |
| 20-40 (공포) | 최대 40% |
| 40-60 (중립) | 최대 60% |
| 60-80 (탐욕) | 최대 80% |
| 80-100 (극단 탐욕) | 최대 50% (과열 주의) |
